# CLAUDE.md — jump-section AI Context

이 문서는 Claude AI가 jump-section 프로젝트를 자율적으로 수정할 때 필요한 컨텍스트입니다.

## 프로젝트 개요

**jump-section**은 웹 페이지에서 메뉴 클릭 시 해당 섹션으로 부드럽게 스크롤 이동하고,
스크롤 위치에 따라 활성 섹션을 자동으로 추적하는 TypeScript 라이브러리입니다.

- **저장소**: https://github.com/bae080311/jump-section
- **npm 패키지**: `@jump-section/core`, `@jump-section/react`, `@jump-section/vue`, `@jump-section/svelte`
- **패키지 매니저**: pnpm + turbo (모노레포)

## 핵심 아키텍처

`ScrollManager` 클래스 (`packages/core/src/ScrollManager.ts`) 가 모든 기능의 중심입니다.
프레임워크 패키지(react/vue/svelte)는 ScrollManager를 래핑한 얇은 어댑터 레이어입니다.

### 주요 API

```typescript
// 옵션
interface ScrollOptions {
  offset?: number;           // 고정 헤더용 오프셋 (px)
  behavior?: ScrollBehavior; // 'smooth' | 'auto' | 'instant'
  hash?: boolean;            // URL 해시 동기화
  root?: HTMLElement | null; // 커스텀 스크롤 컨테이너
  keyboard?: boolean;        // Alt+Arrow 키보드 네비게이션
}

// 핵심 메서드
manager.registerSection(id, element)
manager.unregisterSection(id)
manager.scrollTo(id): Promise<void>
manager.scrollToNext/Prev/First/Last(): Promise<void>
manager.onActiveChange(callback): unsubscribe  // meta: { previous, direction }
manager.onProgressChange(id, callback): unsubscribe
manager.disableSection(id) / enableSection(id)
manager.getSections(): string[]
manager.getActiveId(): string | null
manager.destroy()
```

## 코드 컨벤션

### TypeScript
- **strict mode** 필수 (`tsconfig.json` 참조)
- `any` 타입 사용 금지. 불가피한 경우 주석으로 이유 명시
- Public API는 반드시 TypeScript 인터페이스/타입으로 명시
- optional 필드 추가 시 반드시 기본값 제공

### 포맷팅 (.prettierrc 기준)
- 2 스페이스 들여쓰기, 세미콜론, 싱글쿼트
- `printWidth: 100`, `trailingComma: 'all'`
- 변경 후 `pnpm format` 실행 필수 (CI에서 검사)

### 주석
- Public 메서드에는 JSDoc 한국어 주석 필수: `/** 설명 */`
- 복잡한 로직에는 인라인 주석 (한국어 가능)

## 브랜치 전략

| 패턴 | 용도 |
|------|------|
| `main` | 프로덕션 (push 시 npm publish 트리거) |
| `feat/기능명` | 새 기능 개발 |
| `fix/버그명` | 버그 수정 |
| `ai/issue-{번호}-{설명}` | AI 자동 생성 브랜치 |
| `chore/작업명` | 빌드/설정 변경 |

- main 브랜치 직접 push 금지
- 모든 변경은 PR을 통해 머지
- AI 생성 브랜치는 반드시 사람이 리뷰 후 머지

## 의존성 관리

- `@jump-section/core`: **zero dependency** 유지 필수
- 프레임워크 패키지: 해당 프레임워크만 peerDependency
- devDependency 추가 시 루트 `package.json`에 추가

## SSR 주의사항

- `typeof window === 'undefined'` 체크로 SSR 환경 대응
- DOM API 직접 사용 시 항상 null 체크
- 이벤트 리스너는 반드시 `destroy()` 에서 제거
- scroll 이벤트에 `{ passive: true }` 옵션 필수
